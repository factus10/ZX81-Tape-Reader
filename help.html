ZX81 Tape Reader — User Guide

Overview

ZX81 Tape Reader decodes audio recordings of ZX81, Lambda 8300, and Timex Sinclair 2068 cassette tapes. It analyzes WAV files for frequency-encoded bit patterns and lets you manually review and correct the results before exporting to .tzx or .p format for use in emulators.

Workflow

  1. Open a WAV file via File > Open WAV... (Cmd+O). The tool will analyze the recording for bit patterns. This may take a few seconds depending on file size.
  2. Review the analysis in the editor. Search for ? marks to find areas that need manual correction.
  3. Save your work with File > Save Session (Cmd+S) to create a .ztr session file you can resume later.
  4. Export the final result via File > Export as .tzx... or File > Export as .p...

The Editor Window

The window has three areas:

  • Waveform canvas (top) — shows the original audio waveform and detected frequency bursts. Scroll the waveform by using the mouse wheel over the canvas area. The current editor line is highlighted in blue.
  • Text editor (middle) — the decoded bit data in editable text format. Editing the text immediately updates the waveform display.
  • Status bar (bottom) — shows the total data length in bytes and remaining bits.

Editor Format

Each line in the editor represents one detected frequency burst:

SYMBOL    OFFSET:LENGTH

For example: 0 197441:67

SymbolMeaning
0Zero bit — short pulse matching expected zero length
1One bit — longer pulse matching expected one length
?Unknown — pulse length doesn't match either; needs manual review
-Noise — very short burst, automatically ignored

The offset is the sample index where the burst starts, and length is the burst duration in samples. Both are optional when editing — if omitted, they are inferred from the previous line. This is useful when inserting corrected bits.

Comments can be added after #. The tool automatically adds # suspicious loss of signal? where it detects unusually long pauses in the recording.

Tips

  • Use Cmd+F to search for ? marks and quickly find areas needing attention.
  • Check the byte count in the status bar before exporting. A non-integer number of bytes usually indicates missing or extra bits.
  • When inserting corrected bits, you only need to type the symbol (e.g. 0 or 1). The offset and length will be inferred automatically.
  • Save your session regularly with Cmd+S — tape correction can be time-consuming work.

How It Works

For each sample in the WAV file, the tool uses a Goertzel algorithm to detect 3.2 kHz frequency bursts (the encoding frequency used by the ZX81). Sequences of detected frequencies are collected into runs, and each run's length is compared against the expected lengths for zero and one bits. This frequency-based approach works better than amplitude detection on deteriorated tapes where signal levels vary.
